Análisis de Etiqueta de Rechazo (KO)

Interpretación de parámetros para resolución de Trouble Tickets en S.L.A.M.

Muestra de Etiqueta FAST TRACK
📍

Identificación de Estación (Station ID)

Estructura del código de 4 dígitos (Ejemplo: 3052)

Dígito 1 (Morado)
Planta / Zona Lógica. Identifica la ubicación macro del equipo dentro de RMU1 (ej. Planta 3).
Dígitos 2 y 3 (Verde y Rojo)
SLAM No. Número de la línea S.L.A.M. física (ej. S.L.A.M. 05).
Dígito 4 (Azul)
Printhead No. Número del cabezal impresor o aplicador específico que emitió la etiqueta (ej. Cabezal 2).
⏱️

Trazabilidad de Tiempo

ExSD / CPT Time
Critical Pull Time. Fecha y hora límite en la que el paquete debe estar inyectado en el camión para cumplir la promesa al cliente.
KO Date
Kick-Out Date. Momento exacto en el que el escáner rechazó el paquete y aplicó la etiqueta de error.
⚠️

Diagnóstico de Error (Ticketing)

SP00 Label ID
ScanID. Código de barras único del paquete (SP00...). Indispensable para buscar el histórico en los sistemas internos.
Ticket C/T/I
Category / Type / Item. Ruta de clasificación que S.L.A.M. utiliza para auto-asignar el Trouble Ticket al departamento correcto (ej. Software).
Error Code & Description
Razón del Rechazo. Detalle técnico del fallo. (En la imagen: AutoSlam did not find all the hazmat barcodes required). Falta de lectura de etiquetas de mercancía peligrosa (Hazmat).
